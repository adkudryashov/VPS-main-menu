# 🚀 VPS Server Menu

**VPS Server Menu** — это мощный, удобный и полностью интерактивный Bash-скрипт для быстрого управления и настройки вашего VPS/VDS сервера. 

Забудьте о необходимости запоминать десятки длинных команд. Управляйте прокси, сетью, фаерволом и системными тестами прямо из красивого консольного меню!

---

## ✨ Главные возможности

* 📊 **Системный мониторинг:** Отображение нагрузки на процессор (LA), использования оперативной памяти и аптайма сервера прямо на главном экране.
* 🛡️ **Управление прокси:** Установка и управление панелью **X-UI Pro** ([3x-ui-pro](https://github.com/mozaroc/3x-ui-pro)) — VLESS/Reality/Trojan через nginx с автоматическим SSL и Clash-подпиской. Прямо из меню: патч без потери БД, опциональный AdGuard Home (DNS-over-HTTPS) и бэкап/восстановление.
* ✈️ **Стек telemt / MTProto:** MTProto-прокси для Telegram с **self-SNI маскировкой** — при DPI-пробе отдаёт настоящий сайт панели с валидным сертификатом. В комплекте веб-панель telemt_panel, SYN FIX через MEKO и опциональная пересборка nginx с постквантовым TLS. Подробнее — раздел ниже.
* 🌐 **Управление сетью и IPv6:** Безопасное включение/отключение IPv6 на уровне ядра, проверка доступности и назначенных адресов.
* 🔒 **Фаервол и порты:** Быстрое открытие портов (UFW) и управление SSL-сертификатами (Acme.sh).
* 🧪 **Инструменты тестирования (Menu Tests):** Встроенные скрипты для диагностики сети и обхода блокировок:
  * **Censorcheck** встроен локально (а не скачивается заново при каждом запуске) и поддерживает отдельные режимы: проверка геоблока, радар ТСПУ по сетям РФ (с настройкой собственного RIPE Atlas ключа прямо из меню) или полная проверка.
* 🔄 **Умное обновление:** Встроенная функция `git pull` для обновления меню до последней версии в один клик без перезагрузки сервера.
## 🛠 Установка / Обновление
Запустите команду в терминале вашего сервера:
```bash
bash <(curl -sL https://raw.githubusercontent.com/adkudryashov/VPS-main-menu/main/install.sh)
```

## 🗑 Удаление
```bash
bash <(curl -sL https://raw.githubusercontent.com/adkudryashov/VPS-main-menu/main/uninstall.sh)
```

## ⚙️ Требования
ОС: Ubuntu 20.04+

> Для установки **X-UI Pro** (3x-ui-pro) — отдельное требование апстрима: только Ubuntu 24.04/26.04 или Debian 12/13, без QEMU-эмуляции CPU. Остальные разделы меню от версии ОС не зависят.

## ✈️ Стек telemt / MTProto

Пункт **«Стек telemt / MTProto»** главного меню поднимает MTProto-прокси для
Telegram, замаскированный под настоящий HTTPS-сайт.

### Как работает маскировка

telemt слушает публичный порт и разбирает входящий трафик:

```
Клиент Telegram ──▶ telemt :8444
                       ├─ MTProto-секрет опознан  ──▶ обрабатывает telemt
                       └─ не опознано (DPI-проба) ──▶ сплайс на 127.0.0.1:7444
                                                      (настоящий TLS с валидным
                                                       сертификатом домена панели)
```

Зонд видит обычный веб-сервер, а не прокси. Для этого нужен живой HTTPS —
его даёт уже установленная панель 3x-ui-pro.

### Требования

Стеку нужны **два разных поддомена**, оба с A-записью на IP сервера:

| Переменная | Роль |
|---|---|
| Домен панели | панель 3x-ui-pro + цель self-SNI маскировки |
| Домен REALITY | ключ SNI-роутинга Xray + адрес telemt_panel |

Домены обязаны отличаться: nginx `stream{}` строит `map` по SNI, и одинаковые
значения дают конфликт ключей — `nginx -t` падает с `conflicting parameter`.

### Порты

3x-ui-pro занимает фиксированную внутреннюю топологию, стек её обходит:

| Порт | Кто занимает | Наружу? |
|---|---|---|
| 443 | nginx `stream{}` — SNI-роутер, единственный вход снаружи | да |
| 7443 | панель/подписки, **требует proxy_protocol** | нет |
| 8443 | Xray REALITY inbound | нет |
| 9443 | decoy-сайт домена REALITY | да |
| **7444** | self-SNI vhost для telemt, **без proxy_protocol** | нет |
| **8444** | **telemt** — публичный вход MTProto | да |
| **9444** | **telemt_panel** — веб-UI | да |

Штатный 7443 для маскировки не годится: nginx требует там `proxy_protocol`,
а telemt при сплайсинге этот заголовок не добавляет — соединение рвётся.
Поэтому поднимается отдельный vhost на 7444 с теми же сертификатом и webroot.

### Пункты меню

| Пункт | Что делает |
|---|---|
| Установить весь стек с нуля | Ставит 3x-ui-pro + telemt + telemt_panel. **Стирает существующую панель** — требует ввода слова подтверждения |
| Добавить telemt к существующей 3x-ui-pro | Ставит только telemt и панель, базу 3x-ui не трогает |
| Статус и диагностика | Проверяет службы, наличие vhost маскировки и прогоняет сквозной self-SNI тест |
| Восстановить маскировку | Пересоздаёт vhost из сохранённой конфигурации |
| Показать учётные данные | Читает `/etc/server-menu/telemt-credentials.txt` (права 600) |
| Управление службами | Статус/старт/стоп/логи telemt и telemt_panel |
| MEKO — SYN FIX | Ставит и запускает MEKO, предварительно показав нужные значения и проверив AAAA-записи |
| Пересборка nginx с OpenSSL 3.5 | Постквантовый TLS, 20–40 минут |
| Удалить стек | Убирает telemt и панель, 3x-ui-pro и сертификаты не трогает |

### Важно про обновления панели

Установщик и патч 3x-ui-pro **полностью очищают** `/etc/nginx/sites-enabled`.
Поэтому vhost маскировки лежит в `/etc/nginx/conf.d/telemt-mask.conf` — этот
каталог они не трогают. После любого патча или переустановки панели всё же
прогоните **«Статус и диагностика»**: панель могла перегенерировать пути к
сертификатам, на которые ссылается vhost. Меню X-UI напомнит об этом само.

### Постквантовый TLS (опционально)

Системный OpenSSL на Ubuntu 24.04 — 3.0.x, гибридный обмен ключей
`X25519MLKEM768` не поддерживает. Клиенты iOS его предпочитают, и его
отсутствие — заметный маркер для DPI-эвристик. Пункт пересборки собирает
nginx со своим OpenSSL 3.5.x, не трогая системный (от него зависят apt/ssh).

Скрипт переиспользует **реальные** флаги сборки из `nginx -V`, включая
hardening-опции дистрибутива, и делает бэкап бинарника с автоматическим
откатом и подъёмом сервиса при неудачном `nginx -t`. После пересборки пакет
замораживается (`apt-mark hold`) — security-обновления nginx придётся ставить
вручную, повторив пересборку.

### MEKO не автоматизирован намеренно

MEKO — интерактивный многоуровневый лаунчер, структура меню которого менялась
между версиями. Жёстко прописывать порядок ответов рискованнее, чем пройти
пункты руками. Меню ставит MEKO, показывает нужные значения (порт telemt,
вариант правил `1`) и предупреждает, если у доменов есть AAAA-записи, —
базовая оптимизация MEKO отключает IPv6.
